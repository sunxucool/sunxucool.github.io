    1. URL设计
https://api.chaoxing.com/{VERSION}/{collection}/...

VERSION: v1，v2，v3等版本标示
collection: 同种记录的"集合"

举例：

  - （1）获取学生资源列表

    GET /v1/school/class/users

  - （2）获取学生详细信息

    GET /v1/school/class/{uid} 或 /v1/school/class?uid=1

  - （3）约定过滤字段属性，可<font color="#dd0000">叠加</font>使用

        url?limit=10：指定返回记录的数量
        url?offset=10：指定返回记录的开始位置。
        url?cpage=2&pageSize=100：指定第几页，以及每页的记录数。
        url?sortby=inserttime&order=desc：指定返回结果按照哪个属性排序，以及排序顺序。
        url?param=0：参数指定筛选条件，可根据业务自定义param

返回code状态码说明：

状态码 请求方式 含义和
200 [GET] 服务器成功返回用户请求的数据，该操作是幂等的[Idempotent]
201 [POST/PUT/PATCH] 用户新建或修改数据成功
202 [*] 请求已经进入后台排队（异步任务）
204 [DELETE] 用户删除数据成功。
400 [POST/PUT/PATCH] 请求错误，服务端没有新建或修改数据的操作，该操作是幂等的
401 [*] 表示用户没有权限（Token、用户名、密码错误）。
403 [*] 表示用户得到授权(与401错误相对),但是访问是被禁止的
404 [*] 请求针对的是不存在的记录，服务器没有进行操作，该操作是幂等的
406 [GET] 用户请求的格式不可得（比如用户请求JSON格式，但是只有XML格式）。
410 [GET] 用户请求的资源被永久删除，且不会再得到的。
422 [POST/PUT/PATCH] 当创建一个对象时，发生一个验证错误。
500 [*] 服务器发生错误，无法判断发出的请求是否成功。

返回信息举例：

Code 200:

{
  code:200,//状态码
  message:"",//返回提示信息
  data:[...]//返回数据集合
}

Code 422：

{
  "message": "Validation Failed",
  "errors": [
    {
      "resource": "Issue",
      "field": "title",
      "code": "missing_field"
    }
  ]
}

错误名称 描述
missing 资源不存在
missing_field 缺少必须属性
invalid 请求字段格式无效
already_exists 资源已经存在

YYYY-MM-DDTHH:MM:SSZ

根据请求头限制，该规则由服务端根据请求头的数值自己控制，如下是请求头的描述:

请求头名称 描述
X-RateLimit-Limit 每小时最大的请求数.
X-RateLimit-Remaining 当前窗口时间剩余的请求数.
X-RateLimit-Reset 当前速率限制窗口重置的时间(秒).
